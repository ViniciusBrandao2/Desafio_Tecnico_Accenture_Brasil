# Desafio Técnico – QA Automation | Accenture Brasil

## Sobre o Projeto

Este projeto foi desenvolvido como parte do **Desafio Técnico de QA Automation da Accenture**, com o objetivo de demonstrar conhecimentos em **automação de testes**, organização de código, boas práticas de QA e validação de fluxos funcionais em aplicações web e APIs.
> O uso de **Cucumber/BDD** foi considerado, porém **não foi implementado nesta solução**.

O desafio é dividido em duas partes:
- **Parte 1 – Automação de APIs**
- **Parte 2 – Automação Frontend (UI)**

O projeto foi versionado e entregue via **GitHub**, conforme solicitado.

## Tecnologias Utilizadas

- **JavaScript**
- **Cypress**
- **Testes E2E (Frontend)**
- **Testes de API**
- **Page Object Model (POM)**
- **Git & GitHub**
---

## Estrutura do Projeto

- `cypress/e2e` – Casos de teste automatizados
- `cypress/pages` – Page Objects
- `cypress/fixtures` – Massa de dados e arquivos de apoio
- `cypress/support` – Configurações e comandos customizados
- `cypress.config.js` – Configuração do Cypress

A organização segue boas práticas para facilitar manutenção, leitura e escalabilidade dos testes.

## Considerações Finais

Este projeto demonstra a aplicação prática de:
- Automação de testes funcionais
- Validação de APIs e UI
- Organização de código com Page Objects
- Uso de dados dinâmicos
- Versionamento com GitHub

O foco foi entregar uma solução funcional, organizada e alinhada às boas práticas de QA Automation, conforme proposto no desafio técnico.
